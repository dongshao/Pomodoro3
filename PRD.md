# Pomodoro3 产品需求文档（PRD）

## 1. 文档信息
- 产品名称：Pomodoro3
- 文档版本：v1.0（MVP）
- 文档日期：2026-03-10
- 目标平台：Web（仅前端），部署到 GitHub Pages
- 核心约束：不使用后端服务，不依赖自建 API

## 2. 背景与目标
番茄钟是高频个人效率工具，核心价值在于降低开始任务的阻力，并通过固定专注/休息节奏提升持续工作能力。Pomodoro3 的目标是提供一个可直接在浏览器使用、无登录、无服务端依赖、可快速部署的番茄钟应用。

### 2.1 业务目标
- 在 1 周内交付可上线 MVP。
- 提供可离线使用的基础体验。
- 保证在 GitHub Pages 稳定部署与访问。

### 2.2 用户目标
- 一键开始 25 分钟专注计时。
- 清晰看到当前周期状态（专注/短休息/长休息）。
- 不丢失本地任务和历史数据（在同一浏览器中）。

## 3. 目标用户与使用场景
### 3.1 目标用户
- 学生：需要阶段性专注学习。
- 研发/设计/写作者：需要减少分心，提高深度工作时长。
- 轻量效率工具用户：不想注册、不想同步账号。

### 3.2 典型场景
- 场景 A：用户打开页面后立即开始默认 25 分钟专注。
- 场景 B：用户维护待办任务，完成一个番茄后为任务累积番茄数。
- 场景 C：用户关闭浏览器后再次打开，配置与任务状态仍保留。

## 4. 产品范围
### 4.1 MVP 范围（必须）
- 番茄计时器（专注/短休息/长休息）与自动流转。
- 开始、暂停、重置、跳过当前阶段。
- 今日番茄数统计。
- 任务列表（新增、完成、删除、番茄数累积）。
- 本地数据持久化（`localStorage`）。
- 响铃与浏览器通知（用户授权后）。
- 响应式界面（桌面与移动端）。

### 4.2 非 MVP（暂不做）
- 账号体系与多端同步。
- 团队协作/共享任务。
- 云端报表与多维分析。
- 浏览器插件与桌面端客户端。

## 5. 功能需求
### 5.1 计时器模块
#### 5.1.1 周期规则
- 默认配置：
  - 专注：25 分钟
  - 短休息：5 分钟
  - 长休息：15 分钟
  - 每 4 个专注后进入 1 次长休息
- 用户可在设置中自定义时长和长休息触发间隔。

#### 5.1.2 状态机
- 状态：`idle` / `running` / `paused` / `completed`
- 阶段：`focus` / `shortBreak` / `longBreak`
- 操作：
  - `开始`：从当前剩余时间开始倒计时
  - `暂停`：冻结计时
  - `重置`：重置为当前阶段初始时长
  - `跳过`：立即切换到下一阶段

#### 5.1.3 完成动作
- 阶段结束时播放提示音（可关闭）。
- 若用户允许通知，触发浏览器系统通知。
- 专注阶段结束后，今日番茄数 +1。

### 5.2 任务模块
- 新增任务：输入标题后创建。
- 任务状态：未完成/已完成切换。
- 任务删除：支持删除单个任务。
- 番茄关联：当前专注阶段可绑定到某个任务，完成后该任务番茄数 +1。
- 数据字段：
  - `id`
  - `title`
  - `completed`
  - `pomodoroCount`
  - `createdAt`

### 5.3 统计模块（MVP 轻量版）
- 展示今日完成番茄总数。
- 展示本周完成番茄总数（按本地日期聚合）。
- 不提供图表筛选，仅提供基础数字卡片。

### 5.4 设置模块
- 可配置项：
  - 专注时长
  - 短休息时长
  - 长休息时长
  - 长休息触发周期
  - 自动开始下一个阶段（开关）
  - 提示音（开关）
- 设置变更后立即生效并持久化。

### 5.5 数据持久化
- 使用浏览器 `localStorage`。
- 持久化内容：
  - 用户设置
  - 任务列表
  - 当日/历史番茄计数
  - 当前周期进度（可选：保留剩余秒数）
- 同步策略：前端单机读写，不做跨浏览器同步。

## 6. 非功能需求
### 6.1 性能
- 首屏加载时间目标：< 2 秒（普通网络，GitHub Pages 场景）。
- 计时偏差控制：单次 25 分钟内累计偏差不超过 2 秒（前台标签页）。

### 6.2 兼容性
- 支持最近两个主版本的 Chrome、Edge、Safari、Firefox。
- 移动端至少支持 iOS Safari 和 Android Chrome。

### 6.3 可用性
- 核心操作（开始/暂停/重置/跳过）在一个主视图内完成。
- 按钮和文字在 360px 宽度下可正常显示。

### 6.4 可访问性
- 主要按钮可键盘聚焦和操作。
- 关键文本对比度满足基本可读性。

## 7. GitHub Pages 部署需求
- 项目输出静态资源（HTML/CSS/JS），不依赖服务器渲染。
- 支持部署到仓库 Pages（`gh-pages` 分支或 Actions 工作流）。
- 若使用前端路由，需采用 hash 路由，避免刷新 404。
- 配置仓库 README 中的访问链接与基本使用说明。

## 8. 验收标准（MVP）
### 8.1 功能验收
- 用户可完整跑通一个 4 番茄周期（含长休息）。
- 任务可新增、完成、删除，且刷新后不丢失。
- 今日番茄统计在专注完成后准确增加。
- 设置变更刷新后仍保留。

### 8.2 部署验收
- GitHub Pages 可正常访问且无后端依赖报错。
- 在移动端与桌面端可正常使用核心流程。
- 页面资源加载无阻塞性错误（浏览器控制台无致命报错）。

## 9. 里程碑与排期（建议）
- D1：信息架构与 UI 草图
- D2-D3：计时器状态机 + 基础界面
- D4：任务模块 + 本地持久化
- D5：设置、通知与提示音
- D6：响应式与兼容性修复
- D7：GitHub Pages 部署与验收

## 10. 风险与应对
- 浏览器后台标签页计时不准确：
  - 应对：基于时间戳差值而非纯 `setInterval` 递减。
- 通知权限被拒绝：
  - 应对：保留页面内可见提示，不阻塞核心流程。
- `localStorage` 数据损坏或格式升级：
  - 应对：加版本号与兜底默认值，解析失败自动重置。

## 11. 后续演进方向（MVP 后）
- 番茄历史趋势图（周/月）。
- 自定义主题与白噪音。
- 数据导入导出（JSON）。
- 可选接入云同步（后端版本）。
